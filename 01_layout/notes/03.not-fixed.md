### 前提をそろえたいです

- すでに `.footer` は `position: fixed; bottom: 0` で **画面最下部に固定**されている。
- 新しく `.content-footer` を `main` の中に入れた。
- これを **「画面下」に見せたい**＝「グローバルフッターのすぐ上あたりに、常に出しておきたい」というイメージだと解釈します。

この前提で、「どう固定するか？」の代表的なパターンを整理します。

---

### パターン1：viewport に対して固定（グローバルフッターの“ひとつ上の段”）

**イメージ**  
- `.footer` : 画面最下部（`bottom: 0`）  
- `.content-footer` : その少し上（`bottom: @footer-height;`）  

```less
@footer-height: 56px;
@content-footer-height: 48px;

.app {
  /* 2つ分の高さを確保しておくと、コンテンツが全部見やすい */
  padding-bottom: (@footer-height + @content-footer-height);
  box-sizing: border-box;
}

.content-footer {
  position: fixed;
  /* コンテンツエリアに揃えたい場合は、SideNavの幅ぶんだけ左をずらす */
  left: 200px;  // SideNav=200px のところ。実際はクラス化してあげると◎
  right: 0;
  bottom: @footer-height;          // グローバルフッターの「ちょうど上」
  height: @content-footer-height;  // 必要なら

  border: 1px solid blue;
  background: #fff;  // 見やすくするため
}
```

**ポイント**

- 「画面に対して固定」なので、スクロールしても **常に表示されるバー** になります。
- `.app` の `padding-bottom` を `フッター + content-footer` の合計ぶん持たせると、
  - 最後のコンテンツが両方の裏に隠れず、きれいに収まる  
- SideNav 幅（今は `width: 200px`）を CSS でクラス化しておくと、
  - `left: 200px` のような「マジックナンバー感」が薄れ、保守性も上がります。

**向いているケース**

- 「画面下にいつも出ている CTA / 操作バー」などを作りたい場合
- 「ヘッダーと一緒にスクロールしてほしくない」UI

---

### パターン2：コンテンツ領域の中で固定（コンテンツだけ別スクロール）

こちらは少し性格が違いますが、よくある設計なので参考までに。

**イメージ**

- 右側のコンテンツカラム（SideNav の右側）だけを **独立したスクロールコンテナ** にする
- そのコンテナの中で `.content-footer` を **下に張り付ける**

```less
.app {
  min-height: 100vh;
  padding-bottom: @footer-height;  // グローバルフッターぶん
}

/* main 全体の高さを「ヘッダー＋フッターを引いたぶん」にするなど */
.main {
  height: calc(100vh - @header-height - @footer-height);
  display: flex;
}

/* 右カラムをスクロールコンテナにする */
.main-content {
  flex: 1;
  position: relative;
  overflow: auto;
}

/* コンテナ内の固定フッター */
.content-footer {
  position: absolute;
  left: 0;
  right: 0;
  bottom: 0;

  border: 1px solid blue;
}
```

**ポイント**

- `.content-footer` は **画面に対して固定ではなく、そのカラムの中の「下」に固定**されます。
- ヘッダーと一緒にスクロールしたくない／ページ全体のスクロールを減らしたい UI でよく使われます。

ただし、以前「`main` 自体をスクロールコンテナにしたくない」という話もあったので、  
**今回の要件（画面下に固定）にはパターン1の方が素直**です。

---

### まとめ

- **「画面下に固定」かつ「既にグローバルフッターも固定」なら、実務的には：**
  - `.footer` : `position: fixed; bottom: 0`
  - `.content-footer` : `position: fixed; bottom: @footer-height`
  - `.app`（または `body`）に `padding-bottom: @footer-height + @content-footer-height`  
  という「二段重ねの固定バー」が最もシンプルで壊れにくいパターンです。
- ここでも本質的には「固定要素ぶんの余白を確保する」発想は変わらず、
  - それを **変数（LESS 変数や CSS変数）で一元管理する**
  - どのレイヤー（viewport 寄りか、コンテンツコンテナ寄りか）で固定するかを決める  
  ことで、保守性を高めていく形になります。

もし「画面全幅ではなく、コンテンツ幅にだけ合わせたい」「SideNavがレスポンシブで幅可変」などの条件があれば、そこに合わせた設計パターンも整理できます。
