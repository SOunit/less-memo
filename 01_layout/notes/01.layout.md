結論から言います。

**「ヘッダー・コンテンツ・フッター」で
フッターを下に固定したいレイアウトは、`flex` が最適**です。

grid でもできますが、
**この用途では flex の方がシンプル・安全・意図が明確**です。

---

## なぜ flex がベストか

### 要件を整理すると

- 縦方向のレイアウト
- 上：ヘッダー（高さ固定 or 自動）
- 中：コンテンツ（可変・伸びる）
- 下：フッター（常に最下部）
- **スクロールはコンテンツだけ or 全体**

👉 これは **1次元（縦）レイアウト**
→ **flex の得意分野**

---

## 王道パターン（おすすめ）

### HTML

```html
<body>
  <div class="layout">
    <header>Header</header>
    <main>Content</main>
    <footer>Footer</footer>
  </div>
</body>
```

### CSS / LESS

```less
html,
body {
  height: 100%;
}

.layout {
  min-height: 100vh;
  display: flex;
  flex-direction: column;
}

header {
  flex: 0 0 auto;
}

main {
  flex: 1 1 auto;
  overflow: auto; // コンテンツだけスクロールさせたい場合
}

footer {
  flex: 0 0 auto;
}
```

### これで起きること

- 画面が高い → footer は下に張り付く
- コンテンツが多い → main が伸びてスクロール
- JS 不要
- position: fixed 不要

---

## grid はいつ使う？

grid が向いているのは👇

- ヘッダー＋サイドバー＋コンテンツ＋フッター
- 2次元（行＋列）を同時に扱いたい
- 複雑な画面分割

grid 版の例（参考）：

```less
.layout {
  min-height: 100vh;
  display: grid;
  grid-template-rows: auto 1fr auto;
}
```

👉 これも **問題なく動く**
ただし今回は

- grid のメリットを使い切らない
- flex の方が読みやすい

---

## 「position: fixed」で固定するのは？

### 基本おすすめしない

- スクロールと衝突しやすい
- 高さ計算が必要
- モバイルで事故りやすい

「常に画面に貼り付けたいフッター（広告・操作バー）」
**だけ** に使うべきです。

---

## 結論まとめ

| レイアウト                         | 最適            |
| ---------------------------------- | --------------- |
| 縦1列（header / content / footer） | **flex**        |
| 2次元（sidebarあり）               | grid            |
| 常時表示UI                         | position: fixed |

---

もし次に：

- **フッターは固定＋コンテンツだけスクロール**
- **フッターの高さが可変**
- **モバイルでフッターを隠したい**
- **ゲームUI（ステータスバー付き）**

みたいな条件があれば、
それに合わせた **最適パターン** を出します。
